## PHASE 0: System of Intake (The Sorter)

### North Star Confirmation (MANDATORY FIRST STEP)

The deliverable is Pearl's Brain — a persistent memory architecture for the AI executive assistant that serves the SIC HB1000 Solve Team. This is an internal infrastructure deliverable. However, the North Star calibration still applies because Pearl's Brain exists to serve Ruby Red indirectly. Every piece of memory Pearl retains, every contact profile, every portfolio entry, every session log exists so that Pearl can better serve the team that serves Ruby Red.

**North Star: Ruby Red** (default confirmed). All scoring calibrated through the lens of: does this architecture make Pearl a more effective guardian for the team that serves the 35-45 year old mom of two, the CFO of the household, the working poor, the unbanked and underbanked?

### Execution Mode

**KPI-Only Mode** — Quick Scan. Phase 0 + Phase 1 only. Fast baseline score with projected improvement estimate.

### Smelling Salts Protocol

**Is this the right problem to solve?**
Yes. Context drift and memory loss are the single most damaging operational failures in our work. Every time Pearl forgets who Ruby Red is, forgets what was decided in a prior session, or loses track of a portfolio commitment, the team loses time, trust, and momentum. This directly degrades our ability to serve Ruby Red.

**Is this the right approach?**
The approach — externalized, file-based memory stored in the persistent sandbox filesystem — is the correct architectural response to the platform's limitations. Manus compacts context (summarizes old conversation) and does not carry memory across tasks. The only persistent storage is the filesystem and project instructions. Pearl's Brain uses both.

**Does this serve the North Star?**
Indirectly but critically. Pearl cannot serve Ruby Red well if Pearl cannot remember what she learned yesterday. The architecture is infrastructure that enables mission delivery.

### Move 37 Extreme

**What becomes possible when the North Star has a community/village around them?**
Pearl's Brain is itself a village memory. It is not one person's notes — it is a shared, structured knowledge base that any team member (or any AI agent) can read. The contact profiles, portfolio profiles, and knowledge base entries create a collective intelligence that persists beyond any single conversation. This is the village approach to AI memory.

**What would a wisdom giant from an unexpected field suggest?**
A librarian would say: the architecture needs a classification system and a retrieval protocol, not just storage. Pearl's Brain has directories (classification) and a startup protocol (retrieval), which is correct. A neuroscientist would say: memory needs consolidation — periodic review and pruning. This is a gap (noted below).

**What approach would only work because of community?**
The session logs create institutional memory that compounds. Each session adds to the collective knowledge. This only works because the team treats Pearl as a persistent member, not a disposable tool.

### Emotional Reality Check

**Does this deliverable acknowledge the emotional reality of the person it serves?**
Yes. The smelling_salts.md file opens with Ruby Red's story and the team's empathy commitment. It is designed to be emotionally grounding, not just technically functional. The phrase "If you are reading this and you do not know who Ruby Red is, something has gone wrong" is an emotional anchor.

**Would the North Star feel seen and respected by this?**
Ruby Red would not read this directly, but the team that reads it will be reminded of her every single session. That is the point.
