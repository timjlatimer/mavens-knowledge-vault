# Smelling Salts Protocol — Anti-Drift Mechanism

**Last Updated:** 2026-02-15

The "Smelling Salts Protocol" is a specialized anti-drift mechanism used at two levels within the SIC ecosystem:

**Level 1 — Within the Learning Loop (Phase 0):**
In Learning Loop V13.0, the Smelling Salts Protocol is part of Phase 0 (System of Intake). It challenges the initiative's fundamental assumptions before any work begins:
- Is this the right problem to solve?
- Is this the right approach?
- Does this serve the North Star?

**Level 2 — Pearl's Brain (Persistent Memory):**
The `smelling_salts.md` file in Pearl's Brain is the persistent version of this concept. It contains the core identity, principles, and non-negotiable commitments that Pearl must read at the start of every session. It is the mechanism that prevents Pearl from forgetting who Ruby Red is, what the mission is, and how the team operates.

The name "smelling salts" is deliberately chosen — it is something you use to wake someone up, to bring them back to consciousness. When Pearl starts a new session with no memory of prior work, reading the smelling salts file is the equivalent of being revived.

The user has discussed whether the Smelling Salts Protocol should be integrated into the Learning Loop as a sub-component or kept separate. The recommendation is that it operates at both levels: as a phase within the Learning Loop (already integrated in V13.0) and as a standalone persistent memory mechanism (Pearl's Brain).
