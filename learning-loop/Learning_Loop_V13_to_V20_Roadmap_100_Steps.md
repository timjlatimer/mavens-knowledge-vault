# The 100-Step Journey: Learning Loop V13 to V20

**Prepared for**: Tim HB1000 — Chief Visionary Officer, SIC Solve Team
**Date**: March 20, 2026
**Purpose**: A sequenced roadmap to evolve the Learning Loop from V13 (Prompt-Based Persistence) to V20 (Self-Governing Cryptographic Swarm).

---

## Phase I: The Architecture of Memory (Steps 1-20)
*Closing the gaps of V13 by moving from sandbox files to verifiable state architecture.*

**1. The "Sandbox Escape" Audit** (Research)
Identify every instance where V13 lost context because of platform compaction. Document the failure modes of file-based memory.

**2. State Machine Blueprinting** (Protocol Design)
Draft the JSON schema for a true state machine that survives session termination without relying on the AI to "remember" to read a file.

**3. The "No Paraphrase" Mandate** (Anti-Drift)
Implement a hard rule: AI cannot summarize past decisions. It must quote the exact `CONTEXT_RECORD` ID or halt execution.

**4. Crystallization Engine V1** (Protocol Design)
Build the first automated script that converts conversational history into immutable JSON records.

**5. The Ouroboros Bounding Box** (Governance)
Define the absolute limits of agent self-modification. What code can it rewrite, and what code requires the `1958` passcode?

**6. North Star Compass Calibration** (Ruby Red Alignment)
Separate the Prime Imperative (The Solve Team) from the Local Imperative (e.g., Ruby Red). Formalize the four compass points.

**7. The "Grace Never Judges" Subroutine** (Ruby Red Alignment)
Code the empathy principle into a mandatory API check before any financial recommendation is outputted.

**8. Move 37 Extractor** (Protocol Design)
Build a mechanism to isolate "unconventional" ideas generated during loops and store them in a permanent Pattern Library.

**9. The Orphaned Solutions Scan** (Research)
Formalize the process of scanning external domains (aviation, medicine, law) for governance models we can steal.

**10. Baseline Autonomy Scoring** (KPI Framework)
Implement the 4-dimension autonomy framework (Reversibility, Spend, Relationship Risk, Strategic Alignment) for all agent decisions.

**11. The "Speed Governor" Test** (Anti-Drift)
Replace one written instruction ("Don't skip phases") with one programmatic constraint (API rejects Phase 3 if Phase 2 payload is missing).

**12. The V14 Crystallizer Launch** (Protocol Design)
Deploy the first version of the state-machine protocol. Test it against a complex Maven financial initiative.

**13. The V14 Self-Audit** (Governance)
Run V13 independently against V14 to verify if the Crystallizer actually solved the drift problem or just hid it.

**14. The "Assumption Audit" Gate** (Anti-Drift)
Build the middleware that blocks any AI output that contains a factual claim without a `CONTEXT_RECORD` citation.

**15. Bingo Card Digitization** (KPI Framework)
Convert the conceptual Bingo Card targeting system into a machine-readable JSON array.

**16. The "It's Expensive to be Poor" Metric** (Ruby Red Alignment)
Add a mandatory cost-estimation calculation to every proposed solution. If it costs Ruby Red time or money, it gets flagged.

**17. The Claws Architecture Review** (Research)
Analyze Alex Wissner-Gross's "Claws" model for agent orchestration. Map it to the HB1000 requirements.

**18. The "Three Lines" Defense Model** (Governance)
Implement the IIA Three Lines model: First Line (Self-Enforcement), Second Line (Citation Verification), Third Line (Watchman Audit).

**19. Watchman V2 Deployment** (Governance)
Upgrade the Drift Agent to operate continuously, rather than just at the end of a session.

**20. The "End of Amnesia" Milestone** (Surprise & Delight)
The first time an agent references a decision made 3 weeks ago, by a different agent, in a different session, with perfect fidelity.
