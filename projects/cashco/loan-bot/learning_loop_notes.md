# Universal Learning Loop Protocol (ULL-P) - V3 Notes

## The Loop (3 Steps)

### 1. ASSESS
Score the current version (1-100) on four dimensions:
- **Clarity**: Is it immediately understandable?
- **Completeness**: Does it fully address the requirement?
- **Impact**: Is it compelling and effective?
- **Elegance**: Is it tight — no waste, no bloat?

Total = Average of all four. Then list **Top 3 Weaknesses**.

### 2. ITERATE
Fix the weaknesses aggressively. Document:
- Version number (V1 → V2 → V3…)
- Changes made
- Weaknesses addressed
- New weaknesses introduced (if any)

### 3. EVALUATE → DECIDE
Re-score. Calculate % improvement. Then:
- >10% (Green): Continue — significant gains possible
- 5–10% (Yellow): One more loop — minor polish
- <5% (Red): Ship it — diminishing returns

On Red, deliver: Final version, cumulative gain, and remaining imperfections (ranked).

## Guard Rails
- **Diminishing Returns**: % improvement potential → "Diminishing returns. At [X]%. Ship or pivot."
- **Stuck Loop**: Three iterations each % → "Loop stuck. Consider [alternative approach]."
- **Scoring Integrity**: V1 ≈ 60-70. 85 = excellent. 95+ = rare.

## Session Close
Summary Table with Version, Clarity, Complete, Impact, Elegance, Total, Δ, Signal

One-Line Takeaway: "Next time, I'll [specific action] in my first draft because [what I learned]."
