# Master Lean Canvas Protocol (v4.0 - Manus Exclusive) Notes

## Purpose
This is the "Source Code" for the Cloud Butterfly Lean Canvas system, designed for Manus. It includes a Python Rendering Engine to generate pixel-perfect visual canvases.

## 1. Core Identity: The SOLVE Team
Every canvas must align with three core missions:
1. **Solve Capitalism (The Economy)**: "Cap The Gap" (Blue Seal) — *Purpose with Profit.*
2. **Solve Bureaucracy (The System)**: "Cut The Crap" (Done Seal) — *Outcomes over Process.*
3. **Solve Politics (The Leadership)**: "Track The Pact" (Promise Seal) — *Accountability/Promises Kept.*

## 2. The Two-Page Structure

### Page 1: The Lean Impact Canvas
Standard 9-block Lean Canvas layout, adapted for "Impact" and "Unfair Advantage."

| Block | Description | Key Question |
|:---|:---|:---|
| 1. Problem | Top 3 problems the user faces. | *What is broken?* |
| 2. Customer Segments | Target users and customers. | *Who cares?* |
| 3. Unique Value Prop | Single clear message. | *Why you?* |
| 4. Solution | Top 3 features. | *How do we fix it?* |
| 5. Unfair Advantage | Can't be easily copied/bought. | *What is our moat?* |
| 6. Revenue Streams | Revenue model, life time value, gross margin. | *How do we make money?* |
| 7. Cost Structure | Customer acquisition costs, distribution, hosting, people. | *What does it cost?* |
| 8. Key Metrics | Key activities you measure. | *How do we measure success?* |
| 9. Channels | Path to customers. | *How do we reach them?* |
| 10. Impact | Social/Environmental impact. | *How do we change the world?* |

### Page 2: Strategy Execution Plan (The SOLVE Framework)
Focuses on the "How" and "Why" of execution, explicitly measured against the Three Seals.

| Section | Description | SOLVE Alignment |
|:---|:---|:---|
| Purpose | The "Why" (e.g., "To help people find their second day"). | Blue Seal (Purpose) |
| BHAG | Big Hairy Audacious Goal (10-25 year goal). | Done Seal (Outcome) |
| Core Values | Non-negotiable principles (3-5 max). | Promise Seal (Pact) |
| Quarterly Priorities | Top 3-5 objectives for the next 90 days (Rocks). | Done Seal (Execution) |
| Key Metrics (KPIs) | The numbers we watch daily/weekly. | Blue Seal (Gap Ratio) |
| Strategic Moves | 3-Year and 1-Year strategic targets. | Promise Seal (Accountability) |

## 3. Visual Status System (The "Traffic Lights")
Every block on the canvas MUST have a status indicator.
- 🔴 **RED (Critical)**: Block is undefined, broken, or failing. Immediate attention required.
- 🟠 **AMBER (Caution)**: Block is in progress, partially defined, or at risk. Needs monitoring.
- 🟢 **GREEN (Healthy)**: Block is validated, working, and on target.
- ⚪ **WHITE (Open/Null)**: Block is intentionally left blank or not yet started.

## 4. Dynamic Background Color System
The background color of the canvas visual signals the project's Lifecycle Stage and Urgency.

| Color | Name | Meaning | Protocol |
|:---|:---|:---|:---|
| Blue | Blue Sky | Ideation / Strategic | Use for high-level, long-term, or "blue sky" thinking. Default for new butterflies. |
| Dark Gray | Black Mode | Execution / War Room | Use for "Do or Die" projects. High contrast, high urgency. "War Room" mode. |
| Red | Red Alert | Critical / Crisis | Use when a project is in danger or requires immediate rescue. |
| Green | Green Field | Opportunity / Growth | Use for projects in healthy growth mode, generating revenue. |
| Amber | Amber Warning | Caution / Pivot | Use when a project is stalled or needs a major pivot. |
| Light Gray | White Paper | Archive / Reference | Use for fossilized wisdom, reference libraries, or completed projects. |

**Standard Default:** Blue Sky (unless otherwise specified).

## 5. The Manus Visual Engine (Python Script)
When the user asks for a "Visual Canvas," execute the provided Python code using matplotlib to generate a 2-page lean canvas image.
