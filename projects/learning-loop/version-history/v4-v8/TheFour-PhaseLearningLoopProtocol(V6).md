## Protocol Overview: A Four-Layered System of Quality

The protocol consists of four sequential phases: **Engine A**, **Engine B**, the **Guardian**, and the **Presenter**. This structure is framed using an extended version of Gartner's Pace-Layering model. [4]

| Phase | Engine | Name | Purpose | Key Question |
|---|---|---|---|---|
| **Phase 1** | Engine A | **System of Record** | Ensures the output is **correct** and meets the user's explicit requirements. | *"Is this good enough to deliver?"* |
| **Phase 2** | Engine B | **System of Innovation** | Drives the output to be **competitive** by incorporating external, state-of-the-art best practices. | *"How can we make this even better?"* |
| **Phase 3** | Guardian | **System of Alignment** | Ensures the output has **integrity** and has not drifted from the original user intent. | *"Did we lose the spirit of the request?"* |
| **Phase 4** | Presenter | **System of Communication** | Ensures the output is **communicated** effectively in a presentation-ready format. | *"Is this ready to share?"* |

**A deliverable MUST pass all four phases in sequence before it is delivered to the user.**

## Phase 4: Presenter - System of Communication

### Objective
To create a high-quality, presentation-ready summary of the final, approved deliverable. This ensures that every output is not just correct, competitive, and aligned, but also easily shareable and digestible for a wider audience.

### The Presentation Packaging Process

1.  **Extract Key Insights**: Review the final, approved deliverable (that has passed Phases 1, 2, and 3) and extract the most critical information: the core problem, the key findings, the final recommendations, and the supporting data.

2.  **Outline the Narrative**: Structure the extracted insights into a clear, compelling narrative with a beginning (problem), middle (analysis/findings), and end (conclusion/recommendation).

3.  **Create the Presentation**: Generate a Swiss International Style presentation deck based on the narrative outline. The deck should adhere to the principles outlined in the *Swiss Presentation Design Guide*.

4.  **Final Package**: The final delivery to the user must include:
    -   The final deliverable document (e.g., the report, the analysis, the code).
    -   The comprehensive `LEARNING_LOOP` assessment document.
    -   The Swiss-style presentation deck.

## Protocol Rules & Safety

1.  **Phase Order is Mandatory**: Phase 1 → Phase 2 → Phase 3 → Phase 4.
2.  **Transparency is Key**: All assessments must be documented.
3.  **External Grounding is Not Optional**: The Innovation Scan is required.
4.  **The Guardian is Inviolable**: The Spirit Check cannot be skipped.
5.  **Presentation is Not Optional**: Every final deliverable must be accompanied by a presentation.
6.  **The User is the Final Arbiter**: The user can override any recommendation.
