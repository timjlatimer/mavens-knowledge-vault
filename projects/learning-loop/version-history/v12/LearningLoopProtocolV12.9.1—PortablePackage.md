# Learning Loop Protocol V12.9.1 — Portable Package

## What's In This Package

This package contains the Learning Loop Protocol V12.9.1 formatted for use in multiple AI environments. The protocol is identical across all versions — only the formatting differs to match each platform's requirements.

| File | Platform | How to Use |
|------|----------|------------|
| `learning-loop-protocol-universal-system-prompt.md` | Any LLM | Paste into system prompt or custom instructions |
| `chatgpt-custom-gpt-instructions.md` | OpenAI ChatGPT | Create a Custom GPT → paste into Instructions field |
| `claude-project-instructions.md` | Anthropic Claude | Create a Project → paste into Project Instructions |
| `learning-loop-protocol-v12.9.1-reference.md` | Reference | Complete protocol specification for reading/sharing |

## Quick Start

1. Choose the file matching your AI platform.
2. Copy the instructions section into the appropriate field.
3. Start a conversation with: **"Run the Learning Loop Protocol on [your deliverable]"**
4. Provide your North Star (who the deliverable serves), ethics framework, and intensity when asked.
5. The AI will execute 8 phases with human-in-the-loop checkpoints at each phase.

## Configuration Options

**North Star** (Required): The person or audience the deliverable ultimately serves. Examples: "Ruby Red — mom stretching her budget", "First-time app user, unbanked", "Mid-market sales rep under quota pressure."

**Ethics Framework** (Default: Purpose with Profit):
- Purpose with Profit — Social impact first, financial sustainability second
- Profit with Purpose — Financial viability first, social benefit integrated
- Profit with Profit — Pure financial optimization

**Intensity** (Default: DRIVEN 85%):
- CASUAL (25%) — Light review
- ENGAGED (50%) — Standard review
- DRIVEN (85%) — Deep review with strong adversarial challenge
- PRIMAL (95%) — Maximum scrutiny
- PRIMAL+ (100%) — Nuclear option, everything questioned

## The Eight Phases

| Phase | Name | What It Does |
|-------|------|-------------|
| 0 | System of Intake | Validates the initiative is worth pursuing |
| 1 | System of Record | Establishes baseline quality score (80+ threshold) |
| 2 | System of Innovation | Researches external best practices |
| 3 | System of Adversarial Integrity | Three-persona stress test |
| 4 | System of Communication | Swiss-style presentation output |
| 5 | System of Evolution | Protocol self-reflection and amendments |
| 6 | System of Certification | Final audit and certification (90+ threshold) |
| 7 | Drift Agent | Ongoing alignment monitoring |
