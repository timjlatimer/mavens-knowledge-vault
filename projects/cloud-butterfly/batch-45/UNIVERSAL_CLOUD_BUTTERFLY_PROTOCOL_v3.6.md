# PART 2: WHAT CLOUD BUTTERFLIES ARE

Cloud butterflies are insights generated when a recent memory collides with a long-term memory during unconscious processing (usually overnight). They surface during "Magic Time" — the first 30 minutes after waking — and disappear quickly if not captured.

**THE KEY DISTINCTION:**

| System | What It Harvests | Type |
|--------|------------------|------|
| **Cloud Butterflies** | Unconscious insights | Ideas that surface from pattern collision |
| **PTK (Promises To Keep)** | Conscious commitments | Things you said you'd do |

These are two different capture systems for two different outputs. Both feed into Grace for execution.

| 50-69 | Useful insight with some applications |

# PART 9: STATE PERSISTENCE & CARRY-FORWARD

**CRITICAL:** All AI engines running this protocol MUST track state across sessions and provide explicit carry-forward confirmations.

## 9.1 Status Definitions & Behavior

| Status | What It Means | Carry Forward? | Next Day Behavior |
|--------|---------------|----------------|-------------------|
| **PENDING** | No action taken yet | ✅ YES | Appears in "Accumulated (Previous Days)" queue |
| **APPROVED** | Ready to send | ✅ YES until confirmed sent | Appears in "Approved — Awaiting Send Confirmation" |
| **EDITING** | Needs revision | ✅ YES | Appears in "Needs Edit" section |
| **SNOOZED** | Deliberately delayed | ✅ YES | Reappears in queue with "SNOOZED on [date]" badge |
| **REJECTED** | User doesn't want this | ❌ NO | Removed permanently |
| **COMPLETED** | Action confirmed done | ❌ NO | Archived, removed from active queue |

## 9.2 Required Confirmation Messages

When user takes an action, the AI MUST confirm with explicit carry-forward information:

**On APPROVE:**
```
✅ APPROVED — Email draft ready
📧 From: [SENDER_EMAIL_PRIMARY]
📋 Copy to your email client and send
🔄 Will remain in "Approved" queue until you confirm sent
```

**On SNOOZE:**
```
⏰ SNOOZED — Will reappear tomorrow
📅 This item will appear in Day [X+1] report under "Snoozed Items"
🔄 Snooze date: [today's date]
💡 To act on it sooner, say "show snoozed items"
```

**On EDIT:**
```
✏️ MARKED FOR EDITING
📝 Tell me what changes you want
🔄 Will remain in "Needs Edit" queue until revised
```

**On REJECT:**
```
❌ REJECTED — Removed permanently
🗑️ This item will NOT appear in future reports
```
