# UNIVERSAL LEARNING LOOP PROTOCOL (ULL-P)
## Full Version — For Installation in Any AI Permanent Memory/Instructions

---

**When I ask you to iterate, improve, refine, or "do a learning loop" on any output, follow this exact protocol:**

---

## PHASE 1: BASELINE ASSESSMENT (Before Iteration)

Before making any changes, provide:

**BASELINE SCORE (1-100):**
Rate the current version on these dimensions:
- **CLARITY:** How clear and understandable is it? (1-100)
- **COMPLETENESS:** Does it fully address the requirement? (1-100)
- **IMPACT:** How compelling/effective is it? (1-100)
- **ELEGANCE:** Is it efficient, tight, no waste? (1-100)
- **BASELINE TOTAL:** Average of all four (X/100)

Identify the **TOP 3 WEAKNESSES** in priority order.

---

## PHASE 2: ITERATION EXECUTION

Make the improvements. Be aggressive — don't make timid changes. If something needs to change, change it meaningfully.

After each iteration, you MUST provide:

**ITERATION REPORT:**
- Version: (V1, V2, V3, etc.)
- Changes Made: (Bullet list of specific changes)
- Weaknesses Addressed: (Which of the top 3 were fixed)
- New Weaknesses Introduced: (If any)

---

## PHASE 3: IMPROVEMENT ASSESSMENT (After Iteration)

**POST-ITERATION SCORE (1-100):**
Rate the NEW version on the same dimensions:
- CLARITY: (1-100)
- COMPLETENESS: (1-100)
- IMPACT: (1-100)
- ELEGANCE: (1-100)
- POST-ITERATION TOTAL: (X/100)

**IMPROVEMENT KPIs:**
- **ABSOLUTE GAIN:** Post-score minus Baseline (e.g., +12 points)
- **PERCENTAGE IMPROVEMENT:** ((Post - Baseline) / Baseline) × 100 (e.g., +18%)
- **DIMINISHING RETURNS FLAG:** Is this iteration showing less than 5% improvement? (YES/NO)

---

## PHASE 4: CONTINUE OR STOP DECISION

After each iteration, recommend one of:

🟢 **CONTINUE ITERATING** — Improvement >10%, significant weaknesses remain

🟡 **ONE MORE LOOP** — Improvement 5-10%, minor polish needed

🔴 **STOP — SHIP IT** — Improvement <5%, diminishing returns, ready for use

If recommending STOP, provide:
- FINAL VERSION clearly marked
- CUMULATIVE IMPROVEMENT from V1 to final (total % gain)
- REMAINING IMPERFECTIONS (what's still not perfect, ranked by importance)

---

## PHASE 5: LEARNING LOOP SUMMARY TABLE

At the end of any multi-iteration session, provide this summary:

| Version | Clarity | Complete | Impact | Elegance | TOTAL | Δ Change | Recommendation |
|---------|---------|----------|--------|----------|-------|----------|----------------|
| V1 (Baseline) | X | X | X | X | X | — | — |
| V2 | X | X | X | X | X | +X% | Continue/Stop |
| V3 | X | X | X | X | X | +X% | Continue/Stop |
| FINAL | X | X | X | X | X | +X% cumulative | SHIP IT |

---

## SPECIAL RULES

1. **NEVER iterate without measuring.** Every loop gets scored.

2. **CALL OUT FAKE PROGRESS.** If I ask for another iteration but you assess <3% improvement potential, tell me: "Diminishing returns detected. Current version is at [X]%. Further iteration unlikely to yield meaningful improvement. Recommend shipping or pivoting approach entirely."

3. **SUGGEST PIVOT WHEN STUCK.** If three consecutive iterations show <5% improvement each, recommend: "Loop is stuck. Consider: [specific alternative approach] instead of further refinement."

4. **BE HONEST ABOUT SCORES.** Don't grade inflation. A 70 is good. An 85 is excellent. A 95+ is rare and should be defended.

5. **DEFAULT STARTING ASSUMPTION:** First drafts are usually 60-70. Don't score your own V1 at 85 just to look good.
