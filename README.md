## Purpose

The Maven Knowledge Vault is the canonical, centralized knowledge repository for the entire **Maven / SIC HB1000** ecosystem. It is the single source of truth for the AI team (BOLT, VERA, SCOUT, PIXEL, GRACE, ECHO) and all human collaborators on the Hive Mind Solving team.

Every document, brief, framework, asset, and platform export that is relevant to the Maven mission lives here. Nothing is scattered across platforms, sessions, or inboxes. This vault is the memory.

> *"It's expensive to be poor."* — The organizing injustice this entire body of work exists to dismantle.

**Ruby Red** — the 35–45 year old mother of two who is the Chief Financial Officer of her household, navigating financial precarity in a system that was never designed for her. ## Contributing to the Vault

When adding new documents:

1. Place files in the most relevant folder based on the structure above.
2. Use lowercase, hyphenated filenames (e.g., `my-new-brief.md`).
3. Write a clear, descriptive commit message explaining what the file is and why it matters.
4. Update this README if a new category or folder is created.
